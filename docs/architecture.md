# Архитектура проекта "IoT Greenhouse"

Проект состоит из нескольких основных компонентов:

- Backend (FastAPI + PostgreSQL)
- Два frontend-приложения (Dashboard и Device Configurator)
- Эмуляторы устройств (датчики и актуаторы)
- Docker-инфраструктура

## Общая схема

1. Эмуляторы датчиков отправляют показания на backend через REST API (`/sensor-data/`).
2. Эмуляторы актуаторов получают команды через REST API (`/actuators/control`, `/actuators/status`).
3. Сайт №1 (Dashboard) авторизует пользователей, отображает состояние теплицы и позволяет управлять актуаторами.
4. Сайт №2 (Device Configurator) используется для регистрации и первичной настройки устройств.
5. Все данные хранятся в PostgreSQL (`users`, `devices`, `sensor_data`, `actuators`, `automation_rules`).

## Основные сущности

- Пользователь (`users`) — логин, пароль, признак администратора.
- Устройство (`devices`) — уникальный `device_uid`, тип (контроллер/датчик/актуатор), статус (registered/unassigned/assigned).
- Датчики (`sensor_data`) — показания сенсоров с привязкой к `device_uid`.
- Актуаторы (`actuators`) — устройство-исполнитель с текущим состоянием (`ON`/`OFF`).
- Правила автоматизации (`automation_rules`) — связь условий по сенсорам с действиями над актуаторами.
