# API проекта "IoT Greenhouse"

Ниже приведено краткое описание основных REST-эндпоинтов backend.

## Аутентификация (`/auth`)

- `POST /auth/register` — регистрация пользователя.
- `POST /auth/login` — получение JWT-токена по логину и паролю.

## Устройства (`/devices`)

- `POST /devices/register` — регистрация нового устройства (используется сайтом-конфигуратором).
- `GET /devices/unassigned` — список неустановленных устройств (для администратора на Dashboard).
- `POST /devices/assign` — установка устройства на конкретную локацию (на «доску»).

## Данные датчиков (`/sensor-data`)

- `POST /sensor-data/` — приём новых показаний от датчиков.
- `GET /sensor-data/` — получение списка сохранённых показаний.

## Актуаторы (`/actuators`)

- `POST /actuators/control` — отправка команды актуатору (`ON`/`OFF`).
- `GET /actuators/status` — получение текущего состояния всех актуаторов.

## Автоматизация (`/automation`)

- `POST /automation/rules` — создание правила автоматизации.
- `GET /automation/rules` — получение списка правил.

## Dashboard (`/dashboard`)

- `GET /dashboard/state` — агрегированное состояние теплицы (последние данные сенсоров и статусы актуаторов).
